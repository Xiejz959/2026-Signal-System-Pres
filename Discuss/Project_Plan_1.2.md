# Project Plan 1.2

## 项目背景

这次展示的题目是 **Designing a Noise Reduction Filter for Voice Signals**。

我们的切入点不是直接去处理一个非常复杂的真实环境，而是先用“公共广播 / 站台播报”作为应用背景，再在一个可控的仿真条件下研究语音降噪问题。

具体来说，我们会先准备一段清晰语音，把它看作广播系统中的原始语音信号；然后人工加入高频 hiss 噪声，模拟录音链路或设备中的高频噪声污染；最后设计一个基础滤波系统来改善语音质量。

这样做的好处比较直接：一方面保留了现实应用背景，另一方面也能保证噪声特征可控、实验结果稳定，方便分析和展示。

## 原理说明

从信号与系统的角度看，这个问题本质上是一个信号叠加和系统处理问题。

原始语音和噪声都可以看成信号。当它们叠加以后，得到的就是带噪语音：

`noisy voice = clean voice + noise`

在时域里，语音和噪声混在一起以后并不容易直接区分，所以我们需要借助 **Fourier Transform** 把信号变到频域中去看。频域分析的作用不是直接“降噪”，而是帮助我们看清楚噪声主要集中在哪些频率范围，从而决定滤波器应该怎么设计。

在这次项目里，我们重点考虑的是高频 hiss 噪声。它的特点是高频能量比较明显，而语音中最主要、最影响可懂度的信息通常集中在较低到中等频率范围。因此，从频域上看，语音和高频噪声并不是完全重叠的，这就给滤波提供了可能。

我们的思路就是：先分析 clean voice、noise 和 noisy voice 的频谱分布，再根据这些分布去设计一个基础滤波器，让主要语音成分尽量保留下来，同时把高频噪声压下去。

这里也要明确一点：滤波器并不会“识别语音”，它只是按频率决定哪些成分保留、哪些成分衰减。所以在压制高频噪声的同时，也可能会损失一部分语音高频细节。这也是这个设计里最重要的 tradeoff。

## 解决方案

我们目前采用的主方案是一个可控的仿真流程：

`clean voice -> add simulated high-frequency hiss -> noisy voice -> filter -> filtered voice`

在实现上，我们会先准备一段简短、清晰、适合展示的语音信号。然后用 Python 或 MATLAB 人工生成高频 hiss 噪声，并把它叠加到原始语音上，得到 noisy voice。接下来再设计一个基础的数字滤波器对带噪语音进行处理。

从目前的设定来看，主方案会优先尝试 **basic low-pass filter**。原因很简单：我们的噪声主要集中在高频，低通滤波器在概念上最容易解释，在频域图上也最容易看出作用。如果后续测试发现语音失真太明显，再考虑把 band-pass 作为补充比较方案，但展示主线仍然保持简单清楚。

在 Python 里，低通滤波器并不是一个物理电容，而是一套数字系统规则。它的本质仍然和电路里的 low-pass 一样：低频通过更多，高频衰减更多。不同的是，物理电路靠元件特性完成这个过程，而数字滤波器靠数学系数和信号运算来完成。

## 工作重点

这个项目真正的重点不在“调用了哪个包”，而在于完整完成下面几件事。

第一，先把问题建模清楚。我们处理的不是一个泛泛的“语音降噪”，而是一个高频 hiss 污染下的语音增强问题。

第二，要拿出频域分析结果。也就是说，不只是放处理前后的音频，而是要展示 clean voice、noise 和 noisy voice 的频谱或 spectrogram，让观众看到噪声主要集中在哪一段。

第三，要体现设计过程。我们不能只说“用了一个 low-pass filter”，还要说明为什么选它、cutoff frequency 是怎么考虑的、不同参数可能会带来什么变化。

第四，要把结果讲完整。也就是既讲效果，也讲局限。比如噪声确实减弱了，但语音可能会稍微变闷，这种现象是滤波系统本身带来的，不是错误，而是一种设计权衡。

第五，要始终把项目拉回到 Signals and Systems 的语境里去讲。这里最重要的联系包括：信号叠加、输入输出系统、LTI 系统视角、频率响应，以及 Fourier Transform 对系统设计的指导作用。

## Demo 设计思路

这次 demo 不建议只放音频。最好的方式是把“听到的变化”和“看到的变化”放在一起。

主 demo 推荐使用三段音频：

- clean voice
- noisy voice
- filtered voice

同时配合两类可视化：

- waveform
- spectrogram

这样在展示时，观众既能通过听觉感受到噪声前后差别，也能通过图像看到高频区域发生了什么变化。

推荐的 demo 顺序是这样的：

先用一句话交代背景，比如“我们模拟了一段受到高频 hiss 干扰的公共广播语音”；然后依次播放 clean、noisy 和 filtered 三段音频；最后再展示对应的 waveform 和 spectrogram，对前后变化做解释。

整个 demo 最好控制在一分钟左右，不要做成 live coding，也不要把页面堆得太满。重点是清楚、稳定、直观。

## PPT 结构建议

整个 presentation 可以按 12 页左右来组织：

1. Title and team  
2. Motivation：为什么语音降噪值得研究  
3. Problem statement：输入、输出和目标  
4. Simulated application scenario：为什么用公共广播做背景，为什么采用仿真  
5. Signal characteristics：语音与高频 hiss 的基本特征  
6. Why Fourier Transform helps：为什么要做频域分析  
7. Filter design idea：为什么选择 low-pass，参数怎么考虑  
8. System workflow：从 clean voice 到 filtered voice 的完整流程  
9. Demo setup：语音样本、噪声模型和展示方式  
10. Results：音频前后对比 + waveform + spectrogram  
11. Discussion and limitations：有效性、失真和局限  
12. Conclusion and Q&A

这个结构的重点是：先把问题和原理讲清楚，再做 demo，最后讲结果和局限，这样整个故事会比较完整。

## 执行步骤

第一步，统一背景和术语。也就是说，全组先确定：这次项目的背景就是“仿真的公共广播语音 + 高频 hiss 噪声”，不要在后面来回切换不同场景。

第二步，完成资料查找。重点查三类内容：语音频率特征、高频 hiss 噪声的背景、基础数字滤波器设计。

第三步，完成实验系统。包括准备 clean voice、生成噪声、合成 noisy voice、做 FFT 分析、设计低通滤波器、得到 filtered voice。

第四步，比较参数。至少要看一两组不同 cutoff 或不同设置，说明你们确实做了设计选择，而不是只套一个默认值。

第五步，整理展示材料。输出最终用于课堂的三段音频、waveform 图、spectrogram 图，并挑选最适合展示的一组结果。

第六步，整合 PPT。把背景、原理、流程、结果和局限串成一个完整 presentation，而不是把实验和 slides 分成互不相干的两块。
